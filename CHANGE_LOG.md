## 2026-06-08

- SwiftUI "hello world" skeleton builds and runs on iPhone 15 Pro Max (Developer Mode enabled, device signed and trusted). V1.0 done.
- Project layout: `Relay Notes/Relay Notes/` holds source (`Relay_NotesApp.swift`, `ContentView.swift`, `Assets.xcassets`). Planning docs live at `Relay Notes/planning/` as a folder reference (not a target member).
- **Scope decision:** v1 is voice-to-text only. All LLM-driven cleanup, organization, and categorization moved to "Later." Doc restructured into v1 / Later sections.
- **V1.1 first pass shipped to device.** Mic → AVAudioRecorder (.m4a/AAC 64 kbps mono) → SpeechAnalyzer + SpeechTranscriber (file-based) → SwiftData. Functionally working end-to-end on iPhone 15 Pro Max. Accuracy on "some words" — tuning underway, see "V1.1 accuracy tuning" below.
- **Deviation from plan:** Used `AVAudioRecorder` instead of `AVAudioEngine` for V1.1. Reason: `AVAudioRecorder` writes AAC/m4a natively; `AVAudioEngine → file` requires `AVAudioConverter` middleware. `AVAudioEngine` returns in V1.2 when we need raw PCM buffers for streaming live transcription.
- **No tests yet:** No test target exists. Production code is structured to be testable (DI + protocols). Add a Unit Testing Bundle via Xcode (File → New → Target) before V1.2.
- **Tuning sheet shipped.** All four accuracy knobs are runtime-configurable from the in-app Tuning sheet (slider icon, top-right). `Tunings` (`@Observable`) holds state; `RecorderViewModel` reads a snapshot at the moment of recording/transcribing and passes `RecordingOptions` / `TranscriptionOptions` to the recorder/transcriber. Protocols updated to take options.
- **Tunings persistence shipped.** `Tunings` properties have `didSet` writes to `UserDefaults`; `init` reads back on launch. Settings survive cold starts.
- **V1.1 marked done.** Done-when criterion (speak → stored transcript offline) is satisfied. Accuracy tuning continues in parallel.
- **V1.2 step 1 shipped — notes list.** `ContentView` refactored to a `NavigationStack` with `NotesListView` on top (driven by `@Query` sorted by `createdAt` desc) and `RecorderView` at the bottom (Voice Memos pattern). Tapping a row navigates to a `NoteDetailView` placeholder. Tuning gear moved to the navigation bar.
- **V1.2 step 2 shipped — audio playback in `NoteDetailView`.** New `Audio/AudioPlayer.swift` is a `@MainActor @Observable` wrapper around `AVAudioPlayer`. Detail view shows: timestamp, selectable transcript, then a fixed bottom toolbar with a scrub slider, play/pause button, and elapsed/total time labels. Playback session uses `.playback` category. Player stops on `onDisappear`. Polling loop is a `Task` instead of `Timer` to keep Swift 6 strict concurrency clean.
- **V1.2 step 3 shipped — delete.** `Note.deleteWithAudio(in:)` removes the audio file from disk and the SwiftData row. Wired to swipe-to-delete on the list (no confirmation — swipe gesture is the confirmation) and a trash button in detail view (confirmation alert, since dismissing the screen is a less obvious "you're about to delete" cue). Mode default also changed: `.measurement` → `.default` based on real-world finding that AGC-disabled recordings are too quiet for comfortable playback.
- **V1.2 step 4 shipped — search.** `.searchable` on the navigation stack, in-memory case-insensitive filter on `note.transcript`. Empty-state uses `ContentUnavailableView.search(text:)` to distinguish "no matches" from "no notes." In-memory rather than SwiftData `#Predicate` because `localizedCaseInsensitiveContains` isn't reliably supported in `#Predicate` and at v1 dataset sizes the filter cost is negligible.
- **Debug tooling shipped.** `SampleNotes.seed/deleteAll` and `Tunings.resetToDefaults` exposed as a `#if DEBUG` "Debug" section in the Tuning sheet. Lets us populate the simulator without needing a phone for UX testing.
- **All four V1.2 essentials shipped: list, detail + playback, delete, search.** Daily-driver bar in reach. Three stretch items remain (rename, share, live streaming) — pickable based on what you actually miss in daily use.
- **End-to-end validation in iOS Simulator.** Seeded 12 sample notes via debug action, validated list rendering, search (term match + no-match empty state), swipe-to-delete, trash-button + confirmation flow, detail view (transcript displays; playback shows graceful "couldn't load" error for synthetic notes with no real audio). Console log reviewed — all noise is system/simulator-side (haptic plist absent in simulator, `.searchable` internal constraints, etc.); nothing from our code.
- **V1.2 status:** essentials done. Top-level V1.2 checkbox stays open until daily-use validation on the phone confirms the done-when ("reach for this daily instead of stock Voice Memos"). Stretches are open and prioritization-dependent.
- **Framing pivot — local-first / cloud-opt-in / model-choice axis made explicit.** Doc updated so "offline" and "Apple" are no longer conflated. Three transcriber paths (Apple Speech on-device with no model choice, Local ASR on-device with your-choice models via MLX/whisper.cpp, Cloud STT opt-in only). Same pattern for LLM: local-MLX is the default, cloud is opt-in, Apple Foundation Models are an optional fourth engine (intentionally not primary — Apple is behind the open-source ecosystem on capability as of mid-2026). Pipeline diagram, intro callouts, spine, transcription tier table, inference-engine intro, and architecture notes all updated.
- **Next options:** (a) carry the build on the phone for a day or two and let actual usage decide what stretch to take, (b) pick a small simulator-testable stretch — **Rename** (smallest, real value) or **Share**, (c) bigger: **Live streaming** (`AVAudioEngine` swap-in — phone-only to validate), (d) jump straight to **V1.3 TestFlight ship**.
- **V1.2 stretch shipped — share transcript text.** `ShareLink` added to `NoteDetailView` toolbar (`.topBarTrailing`, beside the trash button), sharing `note.transcript` with `note.displayTitle` as subject (nicer Mail/Messages experience). Disabled when the transcript is whitespace-only. Audio-file sharing intentionally out of scope — text-only was the explicit ask.
- **V1.2 stretch shipped — live streaming transcription.** Architectural swap: `AudioRecorder` (`AVAudioRecorder`) → `LiveAudioEngine` (`AVAudioEngine` + input tap). The tap writes AAC/m4a to disk via `AVAudioFile(forWriting:settings:)` *and* hands PCM buffers (converted via `AVAudioConverter` to the analyzer's `bestAvailableAudioFormat`) into an `AsyncStream`. `Transcriber` protocol gained `makeStreamingSession(options:)`; `AppleSpeechTranscriber`'s session wraps `SpeechAnalyzer.start(inputSequence:)` and exposes a cumulative partial-transcript `AsyncStream<String>`. `RecorderViewModel` has new states (`.recording(partial:)`, `.finalizing`); `RecorderView` renders live partials in a scrollable card above the record button. File-based `transcribe(_:options:)` retained for the future cloud STT path. Validation: builds clean; needs phone-mic testing for real validation.
- **Streaming session now unions `.volatileResults` into user preset.** Phone-test surfaced: with default `.transcription` preset, the live partial card stayed empty (preset emits no volatile/partial results). Switching to `.progressiveTranscription` worked but brings `fastResults` along, costing accuracy. Fix in `AppleSpeechTranscriber.makeStreamingSession` — build the streaming `SpeechTranscriber` from the user's preset's `transcriptionOptions`/`reportingOptions`/`attributeOptions` and *union* `.volatileResults` into reporting options. Live partials regardless of preset; finalized accuracy of the stored transcript unchanged.
- **New planning doc — [planning/transcription-tuning.md](./planning/transcription-tuning.md).** Documents every transcription dial (UI + hidden), what each does, current default + why, plus the streaming-vs-file-based accuracy discussion and a decisions log. The V1.1 accuracy tuning section in notes.md remains the *empirical* record (what we observed when we turned the knobs); the new doc is the *design* record (what the knobs are and why we picked their defaults).
- **V1.2 closed.** All four essentials shipped (list, detail + playback, delete, search) and all three stretches shipped (rename, share, live streaming). Top-level V1.2 checkbox flipped to done. Next: V1.3 — TestFlight ship to personal device.
- **App icon shipped.** Designed via nano-banana (voice→text relay motif: white waveform bars flowing into horizontal text lines on a teal→indigo gradient), composed in Icon Composer, exported as `RelayNotes-Icon.icon`, added to project and wired up via Target → General → App Icon field. System auto-handles Default / Dark / Tinted appearance variants from the single source. Replaces the default empty `AppIcon.appiconset` (still present in `Assets.xcassets`, but ignored).
- **V1.3 reframe.** Original V1.3 was "TestFlight ship to personal device" — but TestFlight requires the paid Apple Developer Program ($99/yr) and the user is on the free tier. Done-when ("home screen + daily use") is actually achievable via Developer Mode sideloading, no enrollment needed. Reframed V1.3 around sideloading-and-dogfooding (7-day re-sign cycle, data persists). Created new V1.4 for TestFlight, explicitly **gated on L1 completion** — committing to the paid program is tied to validating that on-device MLX inference actually works on the phone (concrete capability trigger, not a date or a default). Intro callout (the "Three constraints" `[!info]` block) updated to reflect the new sideload-default framing.
- **Change log split out.** Moved the running narrative from `planning/notes.md` into this file (`CHANGE_LOG.md` at repo root) so `notes.md` becomes the design/plan doc and `CHANGE_LOG.md` is the time-ordered ship record. Added a `CLAUDE.md` rule so future sessions append here after each shippable change.
